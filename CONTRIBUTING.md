# Contributing to ClickerGame

Thank you for your interest in contributing to **ClickerGame**! We
welcome contributions from everyone. This document outlines the
guidelines and expectations for contributing to the project.

------------------------------------------------------------------------

## How to Contribute

There are several ways you can contribute:

### Reporting Issues

-   Use GitHub Issues to report bugs or suggest improvements.
-   Check existing issues before creating a new one to avoid duplicates.
-   Provide clear and detailed descriptions.

### Submitting Pull Requests

1.  Fork the repository.
2.  Create a new branch for your change.
3.  Make your updates.
4.  Ensure your code follows project guidelines.
5.  Submit a pull request with a clear description of your changes.

------------------------------------------------------------------------

## Code Style Guidelines

-   Use consistent indentation (2 spaces for JS/TS, 4 spaces for other
    languages if applicable).
-   Use meaningful variable and function names.
-   Keep functions small and focused.
-   Write clear comments when logic is complex.
-   Follow existing project structure and naming conventions.

------------------------------------------------------------------------

## Testing Requirements

-   Ensure the game runs without errors before submitting.
-   Test new features locally.
-   Verify that existing functionality is not broken.
-   Include steps to reproduce and test your changes in the pull request
    description.

------------------------------------------------------------------------

## Documentation Standards

-   Update documentation when adding new features or changing behavior.
-   Write clear, concise explanations.
-   Use simple language that is easy for new contributors to understand.
-   Keep README and relevant docs up to date.

------------------------------------------------------------------------

------------------------------------------------------------------------

## Proposing New Features

To propose a feature:

1.  Open a GitHub Issue labeled "feature request".
2.  Describe the problem the feature solves.
3.  Provide details about how it should work.
4.  Be open to discussion and feedback.

------------------------------------------------------------------------

## Community Guidelines

We aim to maintain a welcoming and respectful community.

Please:

-   Be respectful and constructive.
-   Provide helpful feedback.
-   Assume good intentions.
-   Avoid harassment, discrimination, or toxic behavior.

------------------------------------------------------------------------

## Questions

If you have questions, feel free to open a discussion or issue in the
repository.

Thank you for helping improve ClickerGame!
